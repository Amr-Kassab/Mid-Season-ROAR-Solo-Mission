# Autonomous Navigation Test

Welcome to the mission! Your task is to write a Path Planner and a Controller to navigate a simulated differential-drive robot from its spawn point `(0.0, 0.0)` to a golden goal sphere located at `(20.0, 0.0)` while avoiding walls.

## The Environment
We provide a simulated backend. When you launch the project, it automatically spins up:
* A static 2D Map (`/map`)
* A TF2 Tree maintaining the robot's physical position (`map` -> `base_link`)
* An RViz visualization window.

## Your Tasks
You need to modify two files located in `src/mission/mission/`:

**1. `planner.py`**
* **Goal:** Implement a discrete search algorithm (e.g., A*, Dijkstra, BFS).
* **Details:** We have already provided the boilerplate to convert the ROS OccupancyGrid into a 2D NumPy array. You must find a path through the zeros (free space) while avoiding the 100s (walls), and use our helper functions to convert that grid path into world coordinates.

**2. `controller.py`**
* **Goal:** Implement a tracking algorithm (e.g., Pure Pursuit, PID).
* **Details:** We provide your robot's current `(x, y, yaw)` and the path generated by your planner. You must compute `v` (linear velocity) and `w` (angular velocity) to make the robot follow the path smoothly.

## Setup & Run Instructions

**Prerequisites:** You must have a working installation of ROS 2 (Humble recommended) and Python 3.

### Step 1: Create Workspace and Clone Repository
Open a terminal, create a new workspace folder, and clone the mission repository directly into it.
```bash
# Create the workspace directory and navigate into it
mkdir -p ~/roar_test_ws
cd ~/roar_test_ws

# Clone the repository directly into this folder
git clone git@github.com:Amr-Kassab/Mid-Season-ROAR-Solo-Mission.git .
```

### Step 2: Install Dependencies
Ensure you have the required ROS 2 dependencies and Python packages installed before building.
```bash
# Update rosdep
rosdep update

# Install ROS 2 package dependencies based on the src folder
rosdep install --from-paths src --ignore-src -r -y

# Ensure NumPy is installed for the planner matrix operations
sudo apt install python3-numpy
```

### Step 3: Build the Workspace
Compile the ROS 2 packages using `colcon`.
```bash
colcon build
```

### Step 4: Source the Environment
You must source the overlay setup file so your terminal recognizes the newly built packages. *(Note: You will need to run this command every time you open a new terminal window).*
```bash
source install/setup.bash
```

### Step 5: Launch the Mission
Launch the simulation, the planner, the controller, and RViz all at once.
```bash
ros2 launch mission start_mission.launch.py env_id:=7
```
*(Tip: Try changing the `env_id` argument from `1` to `10` to test your algorithms against different map layouts!)*
